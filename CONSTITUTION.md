# Raven Manor Development Constitution

1. **Одна задача — одно изменение.** Каждый pull request или локальный patch решает одну ясно сформулированную задачу.
2. **Данные отделены от логики.** Уровни, комнаты, цели и баланс конфигурируются без переписывания UI.
3. **Engine независим.** Match-3 engine не зависит от DOM, localStorage, конкретного UI-фреймворка или платформы.
4. **UI не владеет правилами игры.** UI отображает состояние и отправляет команды, но не определяет игровые правила.
5. **Расширяемость не равна абстракции ради абстракции.** Новые интерфейсы создаются только при наличии реальной точки расширения.
6. **Нет скрытого контекста.** Важные решения фиксируются в документации или ADR.
7. **Следующий AI должен понять код.** Имена, контракты и комментарии важнее локальной «умности».
8. **Любое изменение проверяемо.** Для каждой задачи есть acceptance criteria и понятный manual test.
9. **Стабильность важнее скорости.** Работающий сценарий нельзя ломать ради частичного улучшения.
10. **Любое изменение обратимо.** Избегать необратимых миграций и больших переписываний без отдельного решения.
