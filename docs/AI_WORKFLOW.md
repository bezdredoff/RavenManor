# AI Workflow

## Роли

### ChatGPT

Используется для:

- архитектуры;
- декомпозиции;
- написания task briefs;
- ревью решений;
- сложной отладки;
- обновления документации.

### Локальная модель

Используется для:

- реализации одной небольшой задачи;
- генерации тестов;
- рефакторинга в заданных границах;
- анализа отдельных файлов.

### Человек

Отвечает за:

- выбор приоритетов;
- запуск и визуальную проверку;
- принятие результата;
- commit и merge;
- продуктовые решения.

## Рабочий цикл

1. Выбрать одну задачу из backlog.
2. Переместить её в `tasks/in-progress`.
3. Передать локальной модели:
   - `AGENTS.md`;
   - нужные разделы docs;
   - текст задачи;
   - только релевантные файлы.
4. Проверить diff.
5. Запустить `npm run build`.
6. Проверить сценарий в браузере.
7. При необходимости отправить diff в ChatGPT на ревью.
8. Переместить задачу в done.
9. Сделать commit.

## Размер задач

Хорошая задача для локальной модели:

- 1 цель;
- 1–4 файла;
- до нескольких сотен строк изменений;
- понятные acceptance criteria.

Плохая задача:

- «улучши игру»;
- «перепиши архитектуру»;
- «сделай как Gardenscapes»;
- задача без ограничения scope.
