# FEATURE-013 — Room restoration tasks

## Status

In Review

## Area

Meta

## Goal

Добавить последовательные задания восстановления комнат, которые игрок
выполняет за заработанные в match-3 звёзды.

## User Value

После прохождения уровня игрок получает понятное применение звёздам и видит
непосредственный метаигровой прогресс внутри выбранной комнаты.

## Scope

- по три restoration task для каждой из пяти комнат;
- стоимость и порядок задач;
- чистая логика доступности и завершения задач;
- отображение задач на экране комнаты;
- списание доступных звёзд через стоимость завершённых задач;
- сохранение выполненных задач;
- поддержка существующих сохранений без нового поля;
- unit tests и архитектурная документация.

## Out of Scope

- изменение иллюстрации комнаты;
- альтернативные варианты декора;
- сюжетная сцена после завершения комнаты;
- отдельные счётчики earned/spent stars в сохранении;
- миграционная инфраструктура и versioned saves;
- удалённая конфигурация контента.

## Requirements

- задачи одной комнаты выполняются только по порядку;
- стоимость задачи — положительное количество звёзд;
- доступные звёзды равны заработанным минус стоимость завершённых задач;
- трата звёзд не должна закрывать комнаты или уровни, уже доступные по общему
  количеству заработанных звёзд;
- выполненные задачи сохраняются после перезагрузки;
- старый save без `completedRestorationTasks` продолжает загружаться;
- чистая progression-логика не зависит от DOM или localStorage.

## Acceptance Criteria

- в Вестибюле отображаются три задачи;
- без звёзд первая задача показывает стоимость, но недоступна;
- после получения одной звезды первую задачу можно выполнить;
- после выполнения первой задачи доступные звёзды уменьшаются на один;
- вторая задача становится следующей доступной задачей;
- третья задача остаётся заблокированной, пока не выполнена вторая;
- выполненная задача остаётся выполненной после обновления страницы;
- сброс прогресса удаляет уровни, звёзды и выполненные задачи;
- комнаты открываются по total earned stars, а не по оставшимся звёздам;
- `npm test` проходит;
- `npm run build` проходит.

## Manual Test

1. Сбросить прогресс и открыть Вестибюль.
2. Убедиться, что видны три последовательные задачи.
3. Без звёзд убедиться, что выполнить первую задачу нельзя.
4. Пройти уровень 1 хотя бы на одну звезду.
5. Вернуться в Вестибюль и выполнить «Убрать обломки» за 1 ★.
6. Убедиться, что доступный баланс уменьшился, задача получила статус
   «Выполнено», а следующая стала активной при наличии звезды.
7. Обновить страницу и убедиться, что результат сохранился.
8. Вернуться к списку комнат и проверить, что потраченная звезда не влияет на
   порог открытия комнаты, который считается по заработанным звёздам.
9. Сбросить прогресс и убедиться, что выполненные задачи также сбросились.

## Risks

- изменение стоимости task definitions может изменить вычисляемый баланс
  существующего сохранения;
- FEATURE-015 должна заменить вычисляемый spent balance явным учётом;
- FEATURE-014 должна изменять комнату по completed task IDs, не дублируя
  progression rules в UI.
