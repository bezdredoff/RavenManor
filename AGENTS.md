# Raven Manor — AI Agent Guide

Этот файл обязателен для чтения любым AI перед изменением проекта.

## Цель проекта

Raven Manor — вертикальный mobile-first match-3 с метаигрой восстановления готического особняка.

Текущий приоритет: небольшой, полностью играбельный веб-vertical slice, который легко расширять независимыми AI и разработчиками.

## Перед началом работы

1. Прочитать `docs/PROJECT_CONTEXT.md`.
2. Прочитать `docs/ARCHITECTURE.md`.
3. Открыть конкретную задачу из `tasks/in-progress`.
4. Не менять файлы вне указанного scope без явной причины.
5. Не переписывать работающую систему целиком ради локального изменения.

## Правила реализации

- TypeScript strict mode.
- Данные уровней не должны быть зашиты в UI.
- Match-3 engine не должен зависеть от DOM.
- UI не должен напрямую обращаться к localStorage.
- Новые функции должны сохранять обратную совместимость с существующим прогрессом.
- Предпочитать небольшие модули и явные интерфейсы.
- Не добавлять библиотеку, если задача решается текущим стеком.
- Не менять визуальный стиль без отдельной задачи.

## Формат результата AI

Каждый исполнитель должен вернуть:

1. Краткое описание изменений.
2. Список изменённых файлов.
3. Как проверить результат.
4. Известные ограничения.
5. Риски или последующие задачи.

## Definition of Done

- `npm run build` проходит.
- Нет TypeScript ошибок.
- Основной пользовательский сценарий не сломан.
- Новое поведение можно проверить вручную.
- Данные и UI разделены.
- Документация обновлена, если изменился контракт модуля.
